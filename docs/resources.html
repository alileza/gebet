http/client

http client that can be used for sending http requests and comparing the responses


resources:
    - name: my-awesome-name
      type: http/client
      params:
	  base_url: # (string) base url for the http client that will automatically be prepended to any route in the feature.
	  timeout: # (duration) timeout for the request round-trip.
Actions
  • send
    expressions - $resource sends a $method HTTP request to $target
  • send_body
    expressions - $resource sends a $method HTTP request to $target with payload $body - $resource sends a $method HTTP request to $target with body $body
  • response_code
    expressions - $resource response code should be $code
  • response_body
    expressions - $resource response body should be $body

http/server

http server that mocks API responses


resources:
    - name: my-awesome-name
      type: http/server
      params:
	  port: # (number) http server port to expose
Actions
  • response
    expressions - set $resource response code to $code and response body $body - $resource responds with a $code status code and a response body of $body
  • response_path
    expressions - set $resource with path $path response code to $code and response body $body - $resource with path $path responds with a $code status code and a response body of $body

database/sql

database driver that interacts with a sql database


resources:
    - name: my-awesome-name
      type: database/sql
      params:
	  driver: # (string) sql driver (postgres or mysql)
	  datasource: # (string) sql database source name (`postgres://user:pass@host:port/dbname?sslmode=disable`)
Actions
  • set
    expressions - set $resource table $table list of content $content - $resource $table contains the rows $content - $resource $table contains the row $content
  • check
    expressions - $resource table $table should look like $content - $resource $table should contain the rows $content - $resource $table should contain the row $content

queue

messaging queue that that publishes and serves messages


resources:
    - name: my-awesome-name
      type: queue
      params:
	  driver: # (string) queue driver (rabbitmq)
	  datasource: # (string) queue source dsn (`amqp://user:pass@host:port/`)
Actions
  • publish
    expressions - publish message to $resource target $target with payload $payload - a message is published to $resource target $target with the payload $payload
  • listen
    expressions - listen message from $resource target $target - a listener is bound to $resource target $target
  • count
    expressions - message from $resource target $target count should be $count - there should be $count messages in $resource target $target
  • compare
    expressions - message from $resource target $target should look like $payload
